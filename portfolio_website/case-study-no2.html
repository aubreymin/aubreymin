The U.S Department of Education website is currently unorganized, outdated, and filled with overwhelming information.
We based our persona to a student that is posibbly searching broad information about student loans and grants. Where they can find answers to their questions and presented possible route and options.
Card sorting is a process that we adapt on the user research part of the case study. Card Sorting helped us to organize elements and items on the website and to establish visual hierarchy. To have a better understanding of groups of information, we are able to form a raw map of how we are going to make the new website more human centered.
My project partner and I conducted separate user testings to perform tasks as a student and search for information regarding student loan and grants.
The user was asked to search how to apply for student loan.
The result was succesful, however there were pain points that I have detected in the process:

The user was overwhelmed with the information presented, there was multiple links that made the user undecisive and no visual hierarchy. Lacking of basic information as a summary of the process.
The user was asked to search how to get information about grants for students.
The result was successful, however, same as the first tasks pain points:

The user was overwhelmed with the information, the information is unorganized and the link is taking the user to a different website without warning or confirmation
The user was ask to perform two task
To redesign the website that includes updating and organizing the information. To make it  relatable.
